# AI 기반 맞춤형 뉴스레터 큐레이터
> **정보 과잉 시대, 당신에게 꼭 필요한 뉴스만 AI가 요약해 드립니다.**

## 📺 프로젝트 개요 (Overview)
- **프로젝트 명:** AI 큐레이션 서비스
- **주요 타겟:** 정보 습득 시간을 단축하고 싶은 대학생 및 직장인

## 🚀 개발 배경 및 목적 (Background & Goal)
현대 사회의 정보 과잉 문제로 인해 필요한 정보를 선별하는 데 많은 시간이 소요됩니다. 사용자가 관심 있는 키워드의 뉴스나 기술 블로그를 자동으로 수집하고, 생성형 AI(LLM)를 활용해 핵심 내용을 3줄로 요약함으로써 효율적인 정보 소비를 돕는 것을 목적으로 합니다.

## ✨ 주요 핵심 기능 (Main Features)

1. **🔍 스마트 크롤링:** 설정한 관심 키워드 기반 뉴스 실시간 수집
2. **🤖 AI 자동 요약:** GPT API를 활용한 본문 핵심 3줄 요약 제공
3. **📊 트렌드 대시보드:** 오늘 수집된 정보의 핵심 키워드를 시각화
4. **🔔 맞춤형 알림:** 지정된 시간에 요약된 뉴스레터 푸시 알림
5. **⭐ 지식 아카이브:** 중요한 게시글 스크랩 및 사용자 피드백 기반 추천 최적화

## 3. 형상 식별 방법

### 3.1 버전 규칙: `v주.부.수`
예) `v1.0.0`
* **Major**: 요구사항 변경
* **Minor**: 기능 추가
* **Patch**: 오타 / 경미한 수정

### 3.2 태그 전략

| 기준선 (Baseline)      | 설명      |
| ------------------- | ------- |
| requirements_freeze | 요구사항 확정 |
| design_complete     | 설계 완료   |
| beta_release        | 시제품 완료  |
| final_release       | 최종 제출   |

Git 명령 예: (Commit 후)

* `git tag requirements_freeze`

---

## 4. 변경 관리 절차

### 4.1 변경 요청 절차

1. 변경 필요성 확인
2. 변경요청서 작성 (CR 문서)
3. 별도 브랜치 생성
4. 수정 작업 수행
5. main 브랜치 병합
6. 변경 이력 기록

### 4.2 브랜치 전략

| 브랜치        | 설명      |
| ---------- | ------- |
| main       | 최종본 유지  |
| draft      | 작업 중 문서 |
| review     | 검토 단계   |
| experiment | 실험용 설계안 |

브랜치 생성 예:

* `git branch draft`
* `git checkout draft`

## 5. 형상 통제 방법

모든 변경은 **commit 메시지로 기록**

### commit 메시지 형식

* `[구분자-번호] 변경 내용`

예: * `[CR-01] 요구사항 3번 수정 - 로그인 정책 변경`

### 구분자(prefix)예시

* `[CR]`    Change Request (변경 요청)
* `[FEAT]`  Feature (기능 추가)
* `[FIX]`   Bug Fix (오류 수정)
* `[DOCS]`  Documentation (문서 수정)

➡️ 변경 요청이 있는 경우 CR 번호를 함께 기록:

* `[CR-01]`, `[CR-02]`
* **한 주 최소 1회 commit**
